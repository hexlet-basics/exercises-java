Если переменную можно представить себе как коробку внутри которой лежит некоторое значение,
то массив - это целый ряд коробок, перенумерованных начиная с нуля.
Количество коробок, называемое длиной массива, задается во время объявление и затем не меняется.
Тип значения, лежащего в коробке, тоже определяется во время объявления.
Например для массива целых чисел объявление выглядит так
```java
int[] xxx;
```
Здесь xxx - имя переменной. Оно может быть любым. 
int[] - тип переменной, в данной случае "массив целых чисел". 
Объявление массива строк будет выглядеть так 
```java
String[] xxx;
```
Аналогично будет выглядеть объявление массива с элементами любого другого типа. 
Как и с переменными, после объявления массив должен быть инициализирован. 
Без инициализации его будет невозможно употребить, потому что транслятор будет ругаться 
```bash
variable might not have been initialized
```
Вот объявление и инициализация массива из трех целых чисел
```java
int[] xxx;
xxx = new int[3];
```
Обычно объявление и инициализацию пишут в одну строку
```java
int[] xxx = new int[3];
```
В действительности коробка - это область памяти компьютера и эта инициализация выделит память для трех целых чисел.
Важно, что эта область всегда непрерывна.
Почему это важно?
Доступ к элементам массива осуществляеся по индексу. 
Его еще называют указателем (index - это указатель по латински) 
Так как длины все элементов одинаковые, и они идут друг за другом непрерывно, то зная адрес памяти
для первого элемента можно легко посчитать адрес любого другого элемента зная его индекс.
Поэтому доступ к любому элементу по индексу осуществляеся очень быстро.
Так как нумерация элементов начинается с нуля, то значение первого элемента в массиве - это xxx[0] 
Давайте посмотрим, что у нас лежит в массиве xxx     
Напечатаем первый элемент
```java
System.out.println(xxx[0]);
```
печатает ноль. 
Потому что по умолчанию массив инициализируется нулями для чисел, `false` для boolean[] и `null` для ссылочных типов. 
Если мы хотим инициализировать его, например, числами 2, 3, 5, то это записываеся так 
```java
int[] xxx = new int[]{2, 3, 5};
```
или короче, используя синтаксический сахар 
```java
int[] xxx = {2, 3, 5};

```
У массивов есть метод, дающий его длину
```
xxx.length() для данного массива равно трем
```
Итак
- массив состоит из элементов заданного типа
- занимает непрерывную область памяти
- размер массива не меняется
- любой элемент может быть получен по индексу
