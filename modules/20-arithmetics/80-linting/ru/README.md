Теперь, когда мы уже научились писать простые программы, можно немного поговорить о том, как их писать.

Код программы следует оформлять определённым образом, чтобы он был достаточно понятным и простым в поддержке.

Специальные наборы правил — **стандарты** — описывают различные аспекты оформления кода. В Java одним из самых распространённых считается [Google Java Style](https://google.github.io/styleguide/javaguide.html).

Следить за оформлением вручную неудобно, поэтому используют специальные инструменты. **Линтеры** проверяют код на соответствие стандарту и сообщают о нарушениях, а **форматтеры** автоматически приводят код к нужному виду. В Java для этого есть [google-java-format](https://github.com/google/google-java-format) — он форматирует код по стандарту Google. Взгляните на пример:

```java
System.out.println( "Hello, World!" ); System.out.println("I'm a developer!") ;
```

Здесь сразу несколько отступлений от стандарта:

* лишние пробелы внутри круглых скобок — после `(` и перед `)`;
* лишний пробел перед точкой с запятой;
* две инструкции записаны в одну строку, хотя каждую принято писать с новой.

Форматтер сам исправит оформление и приведёт код к такому виду:

```java
System.out.println("Hello, World!");
System.out.println("I'm a developer!");
```

Соблюдение этих правил не влияет на результат работы программы, но делает код понятнее и проще для восприятия.

Какой мы делаем вывод? Единый стандарт оформления и инструменты, которые его поддерживают, помогают писать код, который легче читать и анализировать.

Помните, что форматтер не отменяет самостоятельного анализа и упрощения кода.
